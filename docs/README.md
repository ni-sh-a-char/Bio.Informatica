## Installation  

> **NOTE** – Bio.Informatica is tested on Linux/macOS (Python 3.9‑3.12). Windows users should use WSL2 or Docker.

### 1. Using Conda (recommended)

```bash
# Create an isolated environment
conda create -n bioinf python=3.11 -y
conda activate bioinf

# Install the core package and all optional scientific dependencies
conda install -c bioconda -c conda-forge bioinformatica
```

*The `bioinformatica` conda package bundles:*

| Dependency | Version (min) | Purpose |
|------------|---------------|---------|
| python | 3.9 | Core language |
| numpy | 1.23 | Numerical arrays |
| pandas | 2.0 | Tabular data |
| biopython | 1.84 | Sequence handling |
| rdkit | 2023.09 | Cheminformatics |
| openmm | 8.0 | Molecular dynamics |
| scikit‑learn | 1.4 | ML utilities |
| matplotlib / seaborn | 3.8 / 0.13 | Plotting |
| jupyterlab | 4.0 | Interactive notebooks |
| click | 8.1 | CLI framework |

### 2. Using pip (for virtualenv or custom builds)

```bash
# Create a virtual environment (optional but recommended)
python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate

# Install the package from PyPI
pip install bioinformatica[all]   # the [all] extra pulls in optional scientific deps
```

> **Tip** – If you need GPU‑accelerated MD (OpenMM) on Linux, install the `openmm-gpu` extra:  
> `pip install "bioinformatica[openmm-gpu]"`

### 3. From source (development mode)

```bash
# Clone the repository
git clone https://github.com/yourorg/Bio.Informatica.git
cd Bio.Informatica

# Install in editable mode with all optional dependencies
pip install -e .[dev,all]
```

*The `dev` extra also installs testing tools (`pytest`, `tox`) and documentation generators (`sphinx`, `mkdocs`).*

### 4. Docker (portable, reproducible environment)

```bash
# Pull the official image
docker pull yourorg/bioinformatica:latest

# Run an interactive container
docker run -it --rm \
    -v "$(pwd)":/workdir \
    -w /workdir \
    yourorg/bioinformatica:latest \
    bash
```

The Docker image contains the full Conda environment, JupyterLab, and the CLI.

## Usage  

Bio.Informatica can be accessed **via a command‑line interface**, **as a Python library**, or **through Jupyter notebooks**. Below are the most common entry points.

### 1. Command‑Line Interface (CLI)

The CLI is built with **Click** and is installed as the `bioinf` entry point.

```bash
# Show top‑level help
bioinf --help
```

#### Sub‑commands

| Command | Description |
|---------|-------------|
| `bioinf seq align` | Pairwise or multiple sequence alignment (uses MAFFT/Clustal‑Omega). |
| `bioinf seq annotate` | Functional annotation via InterProScan or custom HMMs. |
| `bioinf mol dock` | Run a quick docking job (AutoDock Vina wrapper). |
| `bioinf md run` | Launch an OpenMM MD simulation from a PDB or SMILES. |
| `bioinf ml train` | Train a scikit‑learn model on a feature matrix (e.g., QSAR). |
| `bioinf viz plot` | Generate publication‑ready plots (heatmaps, ROC curves, etc.). |
| `bioinf db import` | Import FASTA, SDF, or CSV data into the internal SQLite/Parquet store. |
| `bioinf db query` | Query the internal store with SQL‑like syntax. |

**Example – Align two FASTA files**

```bash
bioinf seq align \
    --method mafft \
    --input data/seq1.fasta data/seq2.fasta \
    --output results/alignment.aln
```

**Example – Run a 10 ns MD simulation**

```bash
bioinf md run \
    --pdb data/1abc.pdb \
    --temperature 310 \
    --steps 5000000 \
    --output results/md_1abc/
```

### 2. Python API  

All functionality is exposed through the `bioinformatica` top‑level package.

```python
>>> import bioinformatica as bi
>>> bi.__version__
'2.4.1'
```

#### Core modules

| Module | Primary objects | Typical use |
|--------|----------------|-------------|
| `bi.sequence` | `FASTA`, `Alignment`, `MotifFinder` | Load, align, and search sequences. |
| `bi.chemistry` | `Molecule`, `DockingEngine`, `ForceField` | Convert SMILES ↔ RDKit, run docking, build OpenMM systems. |
| `bi.md` | `Simulation`, `Trajectory` | Set up, run, and analyse MD trajectories. |
| `bi.ml` | `FeatureExtractor`, `Model`, `CrossValidator` | Build QSAR/ML pipelines. |
| `bi.io` | `DataStore`, `export_*` | Unified I/O (SQLite, Parquet, HDF5). |
| `bi.viz` | `Plotter`, `Heatmap`, `Interactive3D` | Quick visualisation utilities. |

**Simple workflow – Load a FASTA, compute a motif, and plot the result**

```python
from bioinformatica.sequence import FASTA, MotifFinder
from bioinformatica.viz import Plotter

# Load sequences
seqs = FASTA.read("data/viral_genomes.fasta")

# Find the most frequent 6‑mer
motif = MotifFinder(seqs, k=6).most_frequent()
print("Top 6‑mer:", motif)

# Visualise motif frequency across the dataset
Plotter.bar(motif.frequency, title="6‑mer Frequency")
```

### 3. Jupyter Notebooks  

The repository ships with a `notebooks/` directory containing ready‑to‑run examples. Launch JupyterLab with:

```bash
jupyter lab notebooks/
```
